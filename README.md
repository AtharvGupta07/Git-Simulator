# Git Workflow Simulator

A simplified **Git Workflow Simulator** built using **Vanilla JavaScript** to replicate the core Git workflow while emphasizing **Data Structures & Algorithms (DSA)** instead of relying only on DOM manipulation.

## Background

This project is based on a **Frontend Developer assignment from Josh Technology Group**, where I was asked to recreate a Git-like workflow within **2 hours** using only **Vanilla JavaScript**.

The objective wasn't just to build the UI, but to implement the underlying logic in a way that resembles how real applications manage state.

## Goal

The primary focus of this project is to bridge the gap between **visual frontend implementation** and **actual application logic** by:

- Using JavaScript arrays as the source of truth instead of storing state only in the DOM.
- Applying DSA concepts to manage files through different stages.
- Rendering the UI from data rather than treating the DOM as the data store.
- Keeping the implementation closer to how modern frontend applications work.

## Features

- 📁 Working Directory
- 📦 Staging Area
- 📝 Commit History
- ➕ Add new files
- 📌 Stage files
- 🚀 Commit staged files
- 🔄 Dynamic rendering from JavaScript state

## Tech Stack

- HTML
- CSS
- Vanilla JavaScript

## Learning Outcome

This project helped me realize that DSA isn't limited to coding interviews—it has practical use in frontend development as well. Managing application state through data structures instead of directly manipulating the DOM results in cleaner, more scalable, and more maintainable code.
